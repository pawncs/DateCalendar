# D-06: 周视图

## 1. 必要性 (Why)

### 问题
日视图只能看一天，用户需要一个「一周概览」来规划整周的时间安排。周视图是连接日视图（微观）和月视图（宏观，暂不实现）的中间粒度。

### 场景
- 周一规划：打开周视图，将本周待办（todo_week）拖到具体某天
- 时间冲突发现：看到周三已经排满了 3 个会，决定把某个任务挪到周四
- 周末回顾：一目了然本周完成了多少日程

### 与 D-05 日视图的关系
周视图 = 7 个简化的日视图并排。用户可以在周视图中点击某天跳转到日视图。共用同一个 `scheduleStore` 数据源。

## 2. 实现方案 (How)

### 2.1 布局设计

```
┌──────────────────────────────────────────────────────────────┐
│  ← 6月8日 - 6月14日 →          今天               [+ 添加]   │
├──────┬────────┬────────┬────────┬────────┬────────┬──────────┤
│      │ 周一    │ 周二    │ 周三    │ 周四    │ 周五    │ 周六/日  │
├──────┼────────┼────────┼────────┼────────┼────────┼──────────┤
│ 全天  │ 项目启动 │        │        │        │        │          │
├──────┼────────┼────────┼────────┼────────┼────────┼──────────┤
│ 09:00│ 评审会  │        │ 代码审查 │        │ 周会    │          │
│ ...  │        │        │        │        │        │          │
│      │ 待办3项 │ 待办1项 │ 待办5项 │ 待办2项 │ 待办0项 │          │
└──────┴────────┴────────┴────────┴────────┴────────┴──────────┘
```

### 2.2 组件结构

`WeekView.tsx` → `WeekHeader`（周导航 + 7 天日期头）+ `AllDayRow`（全天日程行）+ `WeekGrid`（7 列 × 24 行网格，每列是一个 `DayColumn`）

### 2.3 核心实现

**WeekHeader**：左右箭头切换周，「本周」按钮，7 天日期头（显示星期几 + 日期）。

**DayColumn**：每周中的单列，包含固定时间日程块（绝对定位）+ 底部待办计数。今日列有浅蓝背景高亮。

**日程跨天显示**：`isScheduleOnDay` 函数判断日程是否跨越某天，跨天日程在两天都显示。

**点击跳转**：点击某天日期头 → 跳转到该天的日视图。
