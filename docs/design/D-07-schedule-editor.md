# D-07: 日程创建/编辑

## 1. 必要性 (Why)

### 问题
D-04 提供了后端 CRUD 接口，D-05/D-06 提供了展示视图，但用户还缺一个入口来创建和编辑日程。需要提供多种创建方式。

### 场景
- 从任务详情中关联日程：「这个任务明天下午做」
- 在日视图中点击空白时间段直接创建
- 修改已有日程的时间：「评审会改到明天了」

---

## 2. 实现方案 (How)

### 2.1 三种创建入口

| 入口 | 触发方式 | 说明 |
|------|----------|------|
| 从任务创建 | TaskEditor 中「添加日程」按钮 | 预设 task_id |
| 日视图点击 | 点击时间轴空白处 | 预设 start_time |
| 浮动添加按钮 | 日/周视图顶部的 [+ 添加日程] | 空白创建 |

### 2.2 ScheduleEditor 组件

一个弹窗（Dialog）组件，包含字段：标题、关联任务（可选搜索选择器）、类型（fixed/todo_day/todo_week）、日期/时间选择器、全天开关、颜色选择器。

**根据类型动态切换**：fixed 显示日期+时间范围，todo_day 只显示日期，todo_week 显示周范围。

### 2.3 任务选择器

Popover + 搜索 Input + 任务列表，点击选择关联任务。

### 2.4 日视图点击空白创建

在 ScheduleGrid 上添加 onClick，计算点击位置对应的时间，预设 startTime 后打开 ScheduleEditor。

### 2.5 编辑已有日程

点击日/周视图中的日程块 → 打开 ScheduleEditor（编辑模式），预填所有字段。

## 3. 验证标准 (Verify)

### 功能验证

| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 创建 fixed 日程 | 填写标题+时间 → 保存 → 日视图中显示 | 手动操作 |
| 创建 todo_day 日程 | 选择「今日待办」→ 日视图底部显示 | 手动操作 |
| 创建 todo_week 日程 | 选择「本周待办」→ 周视图中显示 | 手动操作 |
| 关联任务 | 选择关联任务 → 日程与任务关联 | 检查 schedule.task_id |
| 从日视图点击创建 | 点击 15:00 空白 → startTime 预设为 15:00 | 手动操作 |
| 编辑日程 | 点击已有日程块 → 修改标题 → 保存 → 更新 | 手动操作 |
| 全天日程 | 勾选全天 → 不显示具体时间 | 手动操作 |
| 颜色标记 | 选择颜色 → 日程块显示对应颜色 | 手动操作 |
| 表单验证 | 标题为空时保存 → 显示错误提示 | 手动操作 |
