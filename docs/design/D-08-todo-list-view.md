# D-08: 待办列表视图

## 1. 必要性 (Why)

### 问题
日视图和周视图侧重于「时间安排」，但用户有时只需要一个简单的清单：「今天要做哪些事？」「本周还有哪些待办？」。待办列表视图提供了一个更轻量、更聚焦的视角。

### 场景
- 快速打钩：打开待办列表，逐项完成打钩，享受进度条推进的满足感
- 今日聚焦：只看今天要做的待办（todo_day），不被其他日程干扰
- 周清理：查看本周待办（todo_week），把还没做的尽快安排

### 与日/周视图的关系
待办列表视图是日视图/周视图的补充视图。数据来源相同（scheduleStore），只是展示方式不同。用户可以在三个视图之间切换。

## 2. 实现方案 (How)

### 2.1 布局设计

```
┌──────────────────────────────────────────────┐
│  待办列表          [今日] [本周]              │
├──────────────────────────────────────────────┤
│  ┌─ 今日待办 (3/5) ──────────────────────┐  │
│  │  ████████████░░░░░░░░  60%             │  │  ← 进度条
│  │  ✅ 回复客户邮件                        │  │  ← 已完成（划线+灰色）
│  │  ☐ 代码审查           🔴 高优先级       │  │  ← 待完成
│  └─────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### 2.2 组件结构

`TodoListView.tsx` → `ViewToggle`（今日/本周切换）+ `ProgressBar`（完成进度条）+ `TodoItem[]`（待办条目）

### 2.3 核心实现

**TodoListView**：根据 mode（today/week）加载对应日程，过滤出 todo_day/todo_week 类型，计算完成进度。

**TodoItem**：带打钩动画（checkbox scale + 文字划线过渡），关联任务标签，颜色标记圆点，hover 显示删除按钮。

**Progress 组件**：使用 shadcn/ui 的 Progress 组件。

**视图集成**：在 MainLayout 左侧面板添加 Tab 切换（任务树 | 待办列表）。

## 3. 验证标准 (Verify)

### 功能验证

| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 显示今日待办 | 列出所有 todo_day 类型日程 | 手动创建 → 查看待办列表 |
| 显示本周待办 | 列出所有 todo_day + todo_week | 切换「本周」→ 查看 |
| 打钩完成 | 点击待办 → checkbox 打钩 → 文字划线 → 进度条更新 | 手动操作 |
| 取消完成 | 再次点击已完成的待办 → 恢复待办状态 | 手动操作 |
| 进度条 | 完成 3/5 → 进度条显示 60% | 目视确认 |
| 删除待办 | hover → 点删除按钮 → 待办消失 | 手动操作 |
| 空状态 | 无待办时显示引导提示 | 切换到无待办的日期 |
| 视图切换 | 今日 ↔ 本周 → 数据切换 | 手动操作 |
