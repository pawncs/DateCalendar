## 建议执行顺序

```
Phase 0: 多端接入（先行，后续 Phase 可在浏览器中测试完整数据流）
  D-11 (HTTP API 代理 + SQL.js 降级) → D-12 (适配层 + 三种模式)
  → 实现 + 测试 → 后续 Phase 可同时在桌面和浏览器中验证

Phase 2a: 模块二剩余
  D-01 → D-02 → D-03
  (先拖拽排序，再筛选，最后批量操作)

Phase 2b: 模块三日程管理
  D-04 (基石) → D-07 (创建入口) → D-05 (日视图)
  → D-06 (周视图) → D-08 (待办列表)
  → D-09 (状态同步) → D-10 (冲突检测)
```

## 验证方法总览

每个文档的「验证标准」部分包含：
- **功能验证**：具体的测试场景和预期结果
- **交互体验验证**：视觉反馈、动画、无障碍
- **技术验证**：编译通过 + 构建成功 + 手动 E2E

每个功能的测试覆盖三种接入方式：IPC（桌面）、HTTP API（浏览器在线）、SQL.js（浏览器离线）。详见 `docs/test-plans/`。
