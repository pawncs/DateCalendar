# D-05: 日视图

## 1. 必要性 (Why)

### 问题
当前应用只有任务树视图，用户无法看到任务在时间维度上的分布。日视图是日程管理的核心入口——用户每天打开应用，第一眼应该看到「今天有什么安排」。

### 场景
- 早晨规划：打开应用 → 日视图显示今天所有日程
- 时间管理：看到 14:00-16:00 被「项目评审」占用 → 不再安排其他任务
- 进度跟踪：当前时间红线显示现在该做什么，已完成的任务打钩划线

### 设计原则
日视图参考 Google Calendar 的日视图设计，但更轻量：24 小时时间轴左侧，任务块右侧，当前时间红线。

---

## 2. 实现方案 (How)

### 2.1 布局设计

```
┌──────────────────────────────────────────────┐
│  ← 6月10日 (周三)  →          [+ 添加日程]   │  ← 顶部导航
├──────┬───────────────────────────────────────┤
│ 00:00│                                        │
│ ...  │                                        │
│ 09:00│  ┌──────────────────────┐             │
│ 10:00│  │ 项目评审 (fixed)     │             │  ← 固定时间任务块
│ 11:00│  │ 09:30-11:30         │             │
│ 12:00│  └──────────────────────┘             │
│ 14:00│─── 现在 ────────────────────────────   │  ← 红色当前时间线
│ ...  │                                        │
│      │──────── 待办事项 ────────              │  ← todo_day 区域
│      │  ☐ 回复邮件                            │
└──────┴──────────────────────────────────────────┘
```

### 2.2 组件结构

`DayView.tsx` → `DayHeader`（日期导航）+ `TimeAxis`（左侧时间轴）+ `ScheduleGrid`（固定时间日程块）+ `CurrentTimeLine`（红色当前时间线）+ `TodoDayList`（底部待办列表）

### 2.3 核心实现

**TimeAxis**：24 小时 × 60px/h = 1440px 总高，每小时一行。

**ScheduleGrid**：fixed 类型日程用绝对定位，`top` = 开始时间的分钟数，`height` = 持续时间的分钟数。1分钟 = 1px 映射。

**CurrentTimeLine**：红色横线，每分钟刷新位置（`setInterval 60000`）。

**TodoDayList**：在日程网格下方显示 todo_day 类型的待办。

**日期导航**：左右箭头切换日期，「今天」按钮跳回当天。

**滚动到当前时间**：打开时自动 scrollTop 到当前时间上方 200px。
