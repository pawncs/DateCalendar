# D-16: 悬浮窗内容视图

## 1. 必要性 (Why)

### 问题
D-13/D-14 解决了悬浮窗的"壳"，还需要"瓤"——悬浮窗里面展示什么内容。悬浮窗空间有限（340×560），需要精简信息密度，只展示用户最需要快速浏览的内容。

### 场景
- 用户瞄一眼悬浮窗 → 看到今天待办：3 项，其中 1 项已完成
- 用户点击待办项的打钩框 → 完成该项 → 项划掉消失（3 秒后）
- 用户看到接下来 1 小时有"项目评审" → 提前准备
- 用户点击某个待办 → 主窗口打开该任务的详情

### 设计原则
- **极简信息**：不展示完整任务树，只展示与今天相关的日程/待办
- **可操作**：支持打钩完成、快速跳转主窗口
- **紧凑布局**：340px 宽度下信息不拥挤

## 2. 实现方案 (How)

### 2.2 组件结构

```
FloatingContent.tsx
├── FloatingHeader          # 日期 + 星期
├── UpcomingSection         # 接下来 2 小时的固定日程
│   └── UpcomingScheduleItem
├── TodayTodoSection        # 今日待办列表
│   └── TodoItem (打钩动画)
└── WeekOverviewSection     # 本周概览进度条
    └── ProgressBar
```

### 2.3 核心实现

#### 2.3.1 数据来源

复用现有的 `useScheduleStore`，无需新建数据接口：

```typescript
// 今日日程 = fixed 今天 + todo_day 今天
const todaySchedules = useScheduleStore(s => s.daySchedules);
// 本周日程 = 范围查询
const weekSchedules = useScheduleStore(s => s.weekSchedules);
```

悬浮窗加载时自动调用 `loadDaySchedules(today)` 和 `loadWeekSchedules(weekStart, weekEnd)`。

#### 2.3.2 UpcomingSection — 接下来

- 从 `daySchedules` 中筛选 `schedule_type === 'fixed'` 且 `start_time > now`
- 按开始时间升序，最多显示 3 条
- 每条显示：颜色标记条、时间范围、标题
- 点击卡片 → 通过 Tauri 事件通知主窗口打开对应日程

#### 2.3.3 TodayTodoSection — 今日待办

- 从 `daySchedules` 中筛选 `schedule_type === 'todo_day'`
- 按状态排序：未完成在上，已完成在下
- 打钩交互：点击 `☐` → 调用 `updateScheduleStatus(id, 'completed')` → 动画（划线 + 2 秒后淡出）
- 进度显示：`已完成 / 总数`

#### 2.3.4 WeekOverviewSection — 本周概览

- 从 `weekSchedules` 中筛选 `schedule_type === 'todo_week'`
- 计算：进度条 = 已完成数 / 总数
- 显示：`本周待办: N 项 已完成: M 项`
- 进度条颜色：绿色（>50%）/ 橙色（30-50%）/ 灰色（<30%）

#### 2.3.5 与主窗口通信

通过 Tauri 事件系统（`emit` / `listen`）：

| 方向 | 事件名 | 数据 | 用途 |
|------|--------|------|------|
| 主→浮 | `schedule:updated` | `{ schedule }` | 主窗口编辑日程 → 悬浮窗刷新 |
| 主→浮 | `task:updated` | `{ taskId }` | 任务状态变更 → 悬浮窗刷新关联日程 |
| 浮→主 | `focus:schedule` | `{ scheduleId }` | 悬浮窗点击日程 → 主窗口打开编辑 |
| 浮→主 | `focus:task` | `{ taskId }` | 悬浮窗点击任务 → 主窗口跳转 |

### 2.4 空状态处理

| 状态 | 显示内容 |
|------|----------|
| 今日无日程 | 📭 今天没有安排，享受自由时光！ |
| 加载中 | 骨架屏（shimmer animation） |
| 数据加载失败 | ⚠️ 加载失败，点击重试 |

## 3. 验证标准 (Verify)

### 功能验证

| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 显示今日日程 | 有日程时显示在对应区域 | 在主窗口创建今日日程 → 查看悬浮窗 |
| 显示接下来 | upcoming 区域显示开始时间晚于现在的 fixed 日程 | 创建未来日程 → 悬浮窗刷新 |
| 打钩完成待办 | 点击待办 → 划线动画 → 状态同步到数据库 | 点击打钩 → 刷新主窗口确认 |
| 进度条更新 | 完成一个待办 → 进度条实时更新 | 打钩后观察进度条 |
| 空状态 | 今天无日程 → 显示空状态引导 | 切换到无日程的日期 |
| 本周概览 | 进度条显示本周待办完成比例 | 创建多个 todo_week → 完成部分 → 检查 |
| 点击跳转主窗口 | 点击日程 → 主窗口打开对应详情 | 点击后检查主窗口 |

### 交互体验验证

| 场景 | 预期 |
|------|------|
| 打钩动画 | 待办项划线 + 背景变灰 + 2 秒后淡出缩小 |
| 进度条动画 | 完成时进度条平滑增长（transition: width 0.5s ease） |

| 320px 宽度 | 所有内容在 340px 宽度下不溢出、不换行异常 |
