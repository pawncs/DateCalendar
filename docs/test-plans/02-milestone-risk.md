# 测试流程：里程碑与风险

## 前置条件
- 在线模式：运行 `start.bat`（Tauri 桌面应用 + HTTP API :9876 + 浏览器 :5173）
- 离线模式：仅 `npx vite`（SQL.js 降级，数据不持久化）

## 白盒测试（Rust 后端）

```bash
cd datecalendar/src-tauri
cargo test --lib test_update_task_milestone_save test_add_risk test_get_and_delete_risks -- --nocapture
```

### 覆盖用例

| # | 用例 | 验证点 |
|---|------|--------|
| 1 | `test_update_task_milestone_save` | 里程碑设置→持久化→取消 |
| 2 | `test_add_risk` | 添加风险记录 |
| 3 | `test_get_and_delete_risks` | 查询和删除风险 |

## 前端黑盒测试（Playwright）

> 以下用例可在在线模式（HTTP API）或离线模式（SQL.js 降级）中执行。在线模式操作真实数据库，离线模式数据仅存内存。

### TC-01: 设置里程碑

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 选中一个任务 | 右侧显示任务详情面板 |
| 2 | 勾选"标记为里程碑" | 任务标题旁显示星标图标 ⭐ |
| 3 | 点击"保存"按钮 | 里程碑状态持久化保存 |
| 4 | 重新选中该任务 | 里程碑状态保持勾选状态 |

### TC-02: 取消里程碑

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 选中一个已标记里程碑的任务 | 星标显示 |
| 2 | 取消勾选"标记为里程碑" | 星标消失 |
| 3 | 点击"保存" | 取消持久化 |
| 4 | 重新选中 | 里程碑状态保持未勾选 |

### TC-03: 添加风险

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 选中一个里程碑任务 | 显示任务详情 |
| 2 | 切换到"风险"标签 | 显示风险管理面板 |
| 3 | 输入风险描述"进度延迟" | 输入框填充 |
| 4 | 选择概率"高" | 概率设为 high |
| 5 | 输入应对措施 | 应对措施填充 |
| 6 | 点击"添加" | 风险记录创建成功 |

### TC-04: 删除风险

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 选中一个有风险的任务 | 风险列表显示 |
| 2 | 点击删除风险 | 风险记录被移除 |
