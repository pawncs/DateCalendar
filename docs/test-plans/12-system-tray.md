# 测试流程：系统托盘菜单

> 对应设计文档：[D-17 系统托盘菜单](../design/D-17-system-tray.md)

## 前置条件

- 仅桌面模式：运行 `start.bat`（Tauri 桌面应用）
- Windows 系统托盘区域可见（非"隐藏图标"折叠状态可先展开以方便观察）
- 系统托盘为系统级特性，**仅支持手动测试**

## 白盒测试（Rust 后端）

```bash
cd datecalendar/src-tauri
cargo test --lib test_tray_icon_creation test_tray_menu_items test_tray_close_behavior test_quit_via_tray -- --nocapture
```

### 覆盖用例

| # | 用例 | 验证点 |
|---|------|--------|
| 1 | `test_tray_icon_creation` | 应用启动后系统托盘图标成功创建 |
| 2 | `test_tray_menu_items` | 菜单包含"显示主窗口""切换悬浮窗""设置...""退出"四个菜单项 |
| 3 | `test_tray_show_main` | 点击"显示主窗口"→ 隐藏的主窗口恢复显示并聚焦 |
| 4 | `test_tray_toggle_floating` | 点击"切换悬浮窗"→ emit `floating:toggle` 事件 |
| 5 | `test_tray_quit` | 点击"退出"→ `app.exit(0)` 被调用 |
| 6 | `test_close_behavior` | 关闭主窗口 → `CloseRequested` 被拦截 → `window.hide()` |
| 7 | `test_left_click_tray` | 左键单击托盘图标 → 主窗口显示 |
| 8 | `test_tray_tooltip` | 托盘图标 tooltip 为 "DateCalendar" |

## 手动黑盒测试

> 所有用例均需在桌面环境手动执行。

### TC-01: 托盘图标出现与基础信息

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 启动应用 | 系统托盘区域出现 DateCalendar 图标 |
| 2 | 鼠标悬停在图标上 | tooltip 显示 "DateCalendar" |
| 3 | 检查图标清晰度（100% 缩放） | 图标清晰可辨 |
| 4 | 改变系统缩放为 125% 后重启 | 图标仍然清晰 |

### TC-02: 右键菜单结构

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 右键托盘图标 | 弹出系统原生右键菜单 |
| 2 | 检查菜单项 | 显示：显示主窗口、切换悬浮窗、分隔线、设置...、分隔线、退出 DateCalendar |
| 3 | 菜单项数量 | 共 5 个菜单项 + 2 个分隔线 |
| 4 | 点击菜单外区域 | 菜单正常关闭 |

### TC-03: 左键单击托盘 → 显示主窗口

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 关闭主窗口（点 X） | 主窗口隐藏，应用仍在托盘 |
| 2 | 左键单击托盘图标 | 主窗口显示并获取焦点 |
| 3 | 主窗口已显示 → 左键点托盘 | 主窗口聚焦（置前） |

### TC-04: 右键"显示主窗口"

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 主窗口隐藏 | 应用在托盘运行 |
| 2 | 右键托盘 → "显示主窗口" | 主窗口显示并聚焦 |
| 3 | 主窗口已显示 → 右键 → "显示主窗口" | 主窗口聚焦 |

### TC-05: 右键"切换悬浮窗"

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 悬浮窗隐藏 → 右键托盘 → "切换悬浮窗" | 悬浮窗滑出显示 |
| 2 | 悬浮窗显示 → 右键托盘 → "切换悬浮窗" | 悬浮窗隐藏 |
| 3 | 反复操作 5 次 | 每次都能正确切换，不卡顿 |

### TC-06: 右键"退出"

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 右键托盘 → "退出 DateCalendar" | 应用完全退出 |
| 2 | 检查托盘区域 | 托盘图标消失 |
| 3 | 检查任务管理器 | DateCalendar 进程已终止 |
| 4 | 检查悬浮窗 | 悬浮窗也关闭（进程退出后无窗口存在） |
| 5 | 主窗口有未保存数据 | 正常退出，不弹确认窗（设计决定） |

### TC-07: 关闭窗口 ≠ 退出

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 主窗口显示中 → 点击右上角 X | 主窗口隐藏（不是退出） |
| 2 | 检查托盘 | 托盘图标仍存在 |
| 3 | 检查悬浮窗 | 悬浮窗仍可见（或其边缘仍可见） |
| 4 | 按 `Ctrl+Shift+D` | 热键仍可切换悬浮窗 |
| 5 | 按 `Alt+Tab` | 看不到 DateCalendar 主窗口（已隐藏） |

### TC-08: 关闭窗口后恢复 — 完整工作流

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 使用应用（创建任务、查看日程） | 正常使用 |
| 2 | 关闭主窗口 | 主窗口隐藏 |
| 3 | 鼠标靠近右边缘 | 悬浮窗滑出 → 内容仍可查看 |
| 4 | 按 `Ctrl+Shift+D` | 悬浮窗显隐切换正常 |
| 5 | 左键单击托盘图标 | 主窗口恢复，所有数据仍在 |
| 6 | 右键托盘 → "退出" | 应用完全退出 |

### TC-09: 悬浮窗独立于主窗口

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 启动应用 | 主窗口 + 悬浮窗 + 托盘 |
| 2 | 关闭主窗口 | 悬浮窗边缘仍在右边缘 |
| 3 | 触发悬浮窗显示 | 悬浮窗内容正常加载 |
| 4 | 在悬浮窗打钩完成一个待办 | 操作正常，数据更新 |
| 5 | 用托盘恢复主窗口 | 主窗口显示，数据已同步（待办已打钩） |

### TC-10: 边缘场景

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 快速开关主窗口 5 次 | 不崩溃、不卡死 |
| 2 | 退出时悬浮窗正在显示 | 退出后悬浮窗也消失 |
| 3 | 系统托盘图标被折叠到"隐藏图标" | 右键菜单仍可用 |
| 4 | 任务管理器结束进程 | 下次启动能正常恢复 |

## 交互体验验证

| 场景 | 预期 |
|------|------|
| 右键菜单样式 | 使用系统原生右键菜单（与系统其他托盘图标一致） |
| 左键响应 | 单击托盘图标后 500ms 内主窗口显示 |
| 图标清晰度 | 100%/125%/150% 缩放比例下图标不模糊 |
| 退出确认 | 不弹二次确认窗口（菜单操作足够明确） |
| 托盘常驻 | 即使主窗口和悬浮窗都隐藏，托盘图标始终存在 |
