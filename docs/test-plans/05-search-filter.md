# 测试流程：搜索与筛选

## 前置条件
- 在线模式：运行 `start.bat`（Tauri 桌面应用 + HTTP API :9876 + 浏览器 :5173）
- 离线模式：仅 `npx vite`（SQL.js 降级，数据不持久化）

## 白盒测试（Rust 后端）

```bash
cd datecalendar/src-tauri
cargo test --lib test_search_tasks test_search_tasks_case_insensitive -- --nocapture
```

### 覆盖用例

| # | 用例 | 验证点 |
|---|------|--------|
| 1 | `test_search_tasks` | 关键词搜索 |
| 2 | `test_search_tasks_case_insensitive` | 不区分大小写 |

## 前端黑盒测试（Playwright）

> 以下用例可在在线模式（HTTP API）或离线模式（SQL.js 降级）中执行。在线模式操作真实数据库，离线模式数据仅存内存。

### TC-01: 关键词搜索

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 创建任务"前端开发"、"后端API"、"部署上线" | 三个任务显示 |
| 2 | 点击搜索按钮，输入"开发" | 只显示"前端开发" |
| 3 | 清除搜索 | 恢复显示全部 |

### TC-02: 状态筛选

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 创建不同状态的任务 | 任务分布在不同状态 |
| 2 | 点击筛选栏"进行中" | 只显示 status=in_progress 的任务 |
| 3 | 点击筛选栏"已完成" | 只显示 status=completed 的任务 |
| 4 | 点击"全部" | 恢复显示所有任务 |

### TC-03: 优先级筛选

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 创建不同优先级的任务 | 任务分布在不同优先级 |
| 2 | 点击"P2+" | 只显示 priority >= 2 的任务 |
| 3 | 点击"P3+" | 只显示 priority >= 3 的任务 |

### TC-04: 组合筛选

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 同时使用状态筛选 + 优先级筛选 | 同时满足两个条件 |
| 2 | 同时使用筛选 + 搜索 | 三个条件交集 |
| 3 | 清除所有筛选 | 恢复全部显示 |
